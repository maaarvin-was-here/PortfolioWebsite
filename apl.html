Write-up (still in progress!)

At the UW Applied Physics Laboratory, I've been working on a project to design and create an underwater camera capture system to track remote-operated vehicles in a test tank. I've worked on every facet of the project, from modeling physical parts in SolidWorks to writing over 2000 lines of code in Python, using OpenCV computer vision, a Pytorch convolutional neural network, and multi-threading over TCP and UDP networking protocols to bridge an entire system of Raspberry Pi's.

I helped create different models for a Raspberry Pi wall mount which secures the Raspberry Pi and holds the camera towards a set location to make triangulation more accurate. Under the rapid prototyping cycle, I created CAD models of the mounts using SolidWorks and fabricated prototypes using a laser cutter.

A part of the project that I'm especially proud of is a system that I designed to automate data collection and processing. In order to test the delay between various different Raspberry Pi's communicating with the main server, I pointed all the cameras at a phone screen running a stopwatch, and send a threaded request for an image to all the Pi's simultaneously. Using the images that the Pi's send back, I am able to precisely calculate the delay between each Pi.

Instead of manually taking sending the image request and looking at the images that are sent back, I instead implemented two automatic systems: a Pytorch convolutional neural network trained to recognize digits, and an OpenCV Feature Matching method that also finds clusters of pixels that represent digits in an image. Paired with PyAutoGUI, both of these methods are able to convert digits in an image to integers variables, making it extremely easy to find the average delay between Pi's over a large number of trials, a statistic that is crucial to the accuracy and function of the camera system.
